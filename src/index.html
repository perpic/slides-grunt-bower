A package manager for the web

Bower is basically a package manager for the web. It removes the old way of doing things. Oh wait! There is a new version of jquery. Let's go to jquery.com, download the file, copy it into my project folder, wire it and update the version number. What if there is a way to automate all this?

Install packages

Git, HTTP(s), zip, npm


$ bower install [package] --save

$ bower install [package]#[version] --save
Bower can install any registered package, but also runs on Git, HTTP, zip files, and npm. This means [package] is a GitHub shorthand, a Git endpoint, a URL or registry name. [version] can be a version number or range, a git tag or commit, a branch name

Helps run repetitive tasks


Alternative to
Rake/Cake/Make/Ant

Grunt will basically allow you to repeat tasks we have in the Front-end workflow, like: minify files, concatenate them, linting, unit testing... You can consider it an alternative to Rake, Cake, Make, or Ant

package.json + Gruntfile.js

Grunt is very easy to start with. It actually requires just 2 files: package.json and Gruntfile.js

package.json


{
  "name": "slides-grunt-bower",
  "version": "0.0.1",
  "devDependencies": {
    "grunt": "~0.4.5",
    "grunt-contrib-concat": "^0.5.0",
    "grunt-contrib-less": "^0.12.0",
  }
}
						

# Create a basic package.json file
$ npm init

# Install Grunt
$ npm install grunt --save-dev
package.json file includes Grunt, and the list of Grunt plugins you need, plus some metadata. The best way to create a project-specific package.json file is with the command npm init. And to add Grunt or a gruntplugin is to install it via npm with --save-dev option

Gruntfile.js


module.exports = function(grunt) {
  grunt.initConfig({
    pkg: grunt.file.readJSON('package.json'),

    less: { /* Task config */ }
  });

  grunt.loadNpmTasks('grunt-contrib-less');

  grunt.registerTask('default',['less',...]);
};
						

# Run Grunt tasks
$ grunt
$ grunt less
Gruntfile.js is like a command center for everything going on. Here is where you configure your tasks and plugins. It also tell's Grunt to load the plugin with the same name specified in package.json. And finally it registers groups tasks. All tasks can be run alone, like when using grunt less command. grunt command will run less together with any other task grouped under default

Huge ecosystem

One of the really nice things I like so much about Grunt is the huge plugin ecosystem it has. From default tasks already implemented by Grunt team, to any other task you can imagine implemented by others

Watch

Run tasks whenever watched files change

github.com/gruntjs/grunt-contrib-watch / Livereload


$ npm install grunt-contrib-watch --save-dev
						

watch: {
  options: {
    livereload: true,
  },
  less: {
    files: 'src/less/**/*.less',
    tasks: ['less', 'concat']
  }
}
If you need to compile LESS or SASS to CSS, Watch is a must have. Basically it looks for any change on the files you specify, and when it happens, it triggers an action. But there is more. Last version of watch plugin comes with Livereload, which you can enable in the options. This means that everytime watch finishes its actions, Livereload will automatically reload the browser for me - Demo - devtools running?
